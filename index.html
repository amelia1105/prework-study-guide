HTML

  • head element - contains information about the webpage, not visible to users (except for title).
  • body element - visible content shown to the user.
  • attributes - define additional info about an element, always in beginning element
  • title element - required, important for SEO
  • style element - defines CSS styles associated with a website
  • base element - defines base URL for a webpage
  • link element - connects an external resource
  • meta element - charset, http-equiv, viewport
  • child elements - nested within parent elements
  • semantic HTML elements - name of element describes the content and role it plays, use as much as possible
  • non-semantic elements include div and span
  • heading levels h1 to h6, h1 is biggest, should only have one h1 header
  • list elements include ul (unordered lists), ol (ordered), and li (list items)
  • img element - contains info about images on a webpage, attributes include src, alt, width, height
  • src attribute defines location of an image
  • br element - link break
  • a element - anchor, creates links
  • section element - defines sections

CSS

  • 3 ways to style a webpage using CSS: inline, internal style sheet, external style sheet
  • inline - used to style an element directly in HTML file, is less flexible
  • internal style sheet - contains rules for the webpage in the head section of HTML file, rules apply only to that page
  • external style sheet - most common, keep CSS rules in a separate file, makes changes easier. Must use link element in head.
  • margin - indicates how much space we want around the outside of an element (margin=0 -> no space)
  • padding - indicates how much space we want around the content inside an element (increase padding -> increase size of element)
  • selector - all elements on a page will have the declarations
  • combine selectors using commas (h1, h2)
  • img selector - assigns property values to image elements in markup
  • text-align: center declaration will center text in selector
  • display:block declaration makes each image start on a new line
  • rel attribute - specifies relationship between current document and linked document
  • href attribute - specifies location (URL) of external resource, can be a relative path
  • class attribute - used to add styles to certain elements
  • width: 80% means 80% of available width will be used for element
  • margin: 40px auto means 40 px margins top and bottom, auto margins left and right

Git

  • git checkout main - switch to main branch
  • git pull origin main - pull down latest main branch
  • git status - checks what branch you're currently on
  • git checkout -b feature/newbranchname - creates a feature branch and switches to it
  • git add -A - stage code changes, commits all (-A) files in working branch
  • git commit -m "commit message here" - add a commit message
  • git push origin feature/branchname - push feature branch to remote branch

JavaScript

  • control flow - order in which a computer executes code in a script
  • variable - a named container that allows us to store data in our code
  • declaring a variable in JavaScript: use var (old syntax), const, or let
  • JavaScript code lines always end with semicolon
  • console.log method - used to output a message to the web console by adding an argument
  • to link a JavaScript file to an HTML file, use script within the body element of HTML file
  • Chrome DevTools - right click, Inspect, you can write code and test it in Console
  • falsy = 0, -0, On, "", null, undefined, NaN, false
  • truthy = true, all other data types that aren't falsy
  • if statement - only executes if a condition is met
  • strict equality operator (===) checks to see if two values are equal, returns Boolean value
  • array - single variable used to hold group of data, data can be different types
  • index/indices - location of data in an array
  • for loop - performs a task on all items in an array; must have starting point, condition, and iteration statement
  • function - set of instructions that tells the computer how to perform a certain task, must be called to execute
  • function name always followed by open-close parentheses
  • use Math.floor and Math.random to generate random numbers
