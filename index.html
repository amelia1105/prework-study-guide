HTML

  • head element - contains information about the webpage, not visible to users (except for title).
  • body element - visible content shown to the user.
  • attributes - define additional info about an element, always in beginning element
  • title element - required, important for SEO
  • style element - defines CSS styles associated with a website
  • base element - defines base URL for a webpage
  • link element - connects an external resource
  • meta element - charset, http-equiv, viewport
  • child elements - nested within parent elements
  • semantic HTML elements - name of element describes the content and role it plays, use as much as possible
  • non-semantic elements include div and span
  • heading levels h1 to h6, h1 is biggest, should only have one h1 header
  • list elements include ul (unordered lists), ol (ordered), and li (list items)
  • img element - contains info about images on a webpage, attributes include src, alt, width, height
  • src attribute defines location of an image
  • br element - link break
  • a element - anchor, creates links
  • section element - defines sections

Git

  • git checkout main - switch to main branch
  • git pull origin main - pull down latest main branch
  • git status - checks what branch you're currently on
  • git checkout -b feature/newbranchname - creates a feature branch and switches to it
  • git add -A - stage code changes, commits all (-A) files in working branch
  • git commit -m "commit message here" - add a commit message
  • git push origin feature/branchname - push feature branch to remote branch
